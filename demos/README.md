# demos/ — yours

This is **your** space. Every hands-on demo you build during the course lives here, one subfolder per
feature:

```
demos/
  [feature-name]/
    [interface]/   ← e.g. demos/plan-mode/cli/, demos/plan-mode/web/
  <reference-project>/   ← the running project you grow across features
```

Within each feature you'll have **one subfolder per interface** you practiced it on (`cli`, `desktop`,
`ide`, `web`, `mobile`); Web/Mobile demos that make no local files still leave a `NOTES.md`. The
instructor will create demo folders here as you go and remind you that they're yours. Build, break,
and experiment freely — nothing in `demos/` is managed by the instructor.

By the end of the course, the contents of this folder are your **reference project(s)** demonstrating the
Claude features you practiced.
